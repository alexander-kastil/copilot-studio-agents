## Course structure

- **File demos and labs inside the deepest numbered topic subfolder, not the module or sub-module root.** When a module (or a sub-module like `04-ui-update`) has `NN-topic` subfolders, each `demo-NN-*.md` / `lab-NN-*.md` and its companion folder belongs inside the matching topic subfolder. `create-guide` now does this natively (its old "module root" rule was updated). - **Fix links after every move.** Own topic becomes `./`, sibling topics become `../NN-topic/`, and a depth change ripples to cross-tree links (labs reference demos with one extra `../`). ## Slides & decks (pptx)

- **A deck's filename number must match its `demos/NN-*` module folder number.** `01 …` maps to `demos/01-intro`, `02 …` to `demos/02-cowork`, `03 …` to `demos/03-copilot-studio`, `04 …` to `demos/04-maintaining`. The Copilot Studio deck was `02 Implementing…` while its module is `03-copilot-studio`; the user flagged it ("but it is 03-"). Number and title every deck to its module folder.
- **A module's identity lives in three assets that drift independently; reconcile all three.** The master TOC (`demos/readme.md`), the folder guides (`demos/NN-*/readme.md` topic table + topic H1s), and the pptx deck (filename + title slide + agenda + section dividers). The folder is the source of truth; align the TOC and deck to it. Keep a topic title identical across the TOC bullet, the module-table row, the topic H1, and the deck divider/agenda.
- **When one deck carries two modules, split it one-deck-per-module.** Extract the module's section into that module's own deck, mirroring the folder separation (the Publishing & Monitoring section moved out of the Copilot Studio deck into a Publishing & Maintaining deck).
- **Edit existing branded decks in place with python-pptx; never regenerate from scratch (that discards the branding).** The proven recipe is in the global `create-slides` skill (`references/edit-existing-pptx.md`): `PYTHONUTF8=1` plus `sys.stdout.reconfigure(encoding='utf-8')` to dodge the Windows cp1252 crash on zero-width chars; address slides by live order or title (part-file numbers resequence on save, so never trust `slideN.xml` == position across a save); delete via `sldIdLst.remove` + `drop_rel`; clone a slide by deep-copying shapes off a same-layout text-only template with rId remap; grow a bullet list by deep-copying the template `<a:p>`; split a deck by copy-then-keep-only (avoids fragile cross-package slide copy). No LibreOffice here, so verify new slides structurally (title, bullets, geometry within bounds) and tell the user no visual render ran.
- **Two python-pptx gotchas that cost a redo this session (now in the global `edit-existing-pptx.md`):** (1) when repositioning a placeholder, set ALL of left/top/width/height; setting only left/width writes the rest as 0 and collapses the body to the top, overlapping the title (this is exactly the Cowork overlay bug). (2) A "text-only" clone template must have no picture PLACEHOLDER (`PP_PLACEHOLDER.PICTURE`, whose `shape_type` is PLACEHOLDER, not PICTURE); a filled one repeats the same image on every clone and makes the deck useless. Geometry numbers hide overlap, so **render to verify**: `winget install TheDocumentFoundation.LibreOffice` + `pip install pymupdf`, then pptx to pdf (soffice) to png (fitz). Never claim a slide looks right without rendering. **On Windows with Office, PowerPoint COM (win32com) is the zero-install, true-fidelity renderer:** `pres.Slides(i).Export(path, "PNG", 1600, 900)`; the Export call fails on forward-slash / 8.3 short-name paths, so pass `os.path.normpath(os.path.abspath(...))`. A rendered pass confirmed the Cowork overlay fix and all five native diagrams look correct.
- **To merge slides ACROSS decks with fidelity, use PowerPoint COM, not python-pptx.** `python-pptx` cannot copy a slide faithfully (picture/table relationships break). When Office is installed (Windows), `win32com.client.Dispatch("PowerPoint.Application")` then `pres.Slides.InsertFromFile(source, Index, SlideStart, SlideEnd)` inserts source slides `SlideStart..SlideEnd` (1-based) *after* position `Index`, preserving layouts, images, and tables and remapping media automatically; `SaveAs(path, 24)` writes .pptx. This beats the ZIP/XML `merge-slides.md` script when you need a *subset* of slides or a *specific insert position* (that script is append-only, cross-platform). Both paths and a decision table are now in `create-slides/references/merge-slides.md` (local == global). **"Unify two decks, take the best of both" workflow:** list both decks' slide titles, insert only the source slides whose topic the base lacks, at the topically correct position; drop redundant title/summary/duplicate slides. This session merged 3 unique admin-center slides (Teams AC, PP AC, PP AC Areas) from a 5-slide governance draft into the 33-slide maintaining deck at position 10, then deleted the two source decks only after explicit user confirmation.

## Copilot Studio agent YAML (`src/agents/**/*.mcs.yml`)

> Distributed into the local `copilot-studio` skill: verification ladder -> `references/schema-verification.md`; entity catalog -> `references/prebuilt-entities.md`; card contract -> `references/adaptive-cards.md`; flow/scoping -> `references/conversation-design.md`; file anatomy -> `references/topic-yaml.md`; extension workflow -> `references/clone-and-sync.md`. Entries below are the source log.

- **Microsoft Learn documents the Copilot Studio designer UI and the concepts, but almost never the literal YAML token.** The entities page gives display names ("Date and time") and variable base types and stops there. Asking Learn "what is the YAML identifier" is a dead end, and web search actively misleads: searching the date entity returned confident claims for both `DatePrebuiltEntity` and `DateTimePrebuiltEntity`. Both strings exist and mean different things.
- **The authoritative oracle is local: the language server binary shipped with the VS Code extension.** `grep -aoE '[A-Za-z]+PrebuiltEntity' ~/.vscode/extensions/ms-copilotstudio.vscode-copilotstudio-*/lspOut/LanguageServerHost.exe | sort -u` prints the complete valid token table. `grep -a` is required (without it grep says "Binary file matches" and prints nothing), the version suffix must be globbed, and the tokens are ASCII (a UTF-16 decode finds nothing). The same binary's `AdaptiveCard*` diagnostic names double as a requirements checklist: `AdaptiveCardMissingActionSubmit`, `AdaptiveCardInputIsRequiredMissingLabel`, `AdaptiveCardInputIsRequiredMissingErrorMessage`. Reading the error catalog beats discovering rules by trial.
- **Absence from the binary is not proof of invalidity.** `NumberPrebuiltEntity` appears only in generated `Read*`/`Write*`/`Visit*` visitor forms, yet works in two topics here. Confirm against a working file before rejecting a token.
- **`.mcs/botdefinition.json` is NOT independent confirmation of a token.** It embeds the topic YAML verbatim as an escaped string, so a token found there only proves the same local `.yml` contains it, never that Copilot Studio accepted it. Treat the bundle and the `.yml` as one source.
- **Verify every `.mcs.yml` edit with the language server, do not eyeball it.** `mcp__ide__getDiagnostics` with no `uri` returns all analyzed files; find the topic and expect `"diagnostics": []`. Passing a `uri` for an unopened file returns an empty list that only means "not analyzed", which reads as a false pass. Fall back to the VS Code Problems pane (`Ctrl+Shift+M`) when the IDE MCP is disconnected (it was, mid-session).
- **The `microsoft-learn` MCP in `.mcp.json` was not connected this session**; only an OAuth-gated `claude.ai Microsoft Learn` connector was available, which needs the user to run `/mcp`. When the user asks to "check against the MS Learn MCP" and it is not up, say so and name the fallback rather than silently substituting web search.
- **A `Question` node cannot collect Adaptive Card input; `AdaptiveCardPrompt` is the interactive node.** Its contract is three parts that must agree: the card JSON input `id`, `output.binding` mapping that id to a variable, and `outputType.properties` declaring its type. Default the card `version` to `1.5`, not 1.6: Teams and the Omnichannel widget cap at 1.5, and 1.6 renders only in the test chat, never on the authoring canvas.
- **Swapping a Question for a card trades away NLU and slot filling.** A card input stores the typed value verbatim; there is no entity extraction and no proactive slot filling. Right trade for a multi-field form, wrong one for a single conversational value. Say so when making the swap instead of presenting it as a pure upgrade.
- **Match the surrounding file on style, do not normalize to an external sample.** External guides claim `variable: init:Topic.X` is mandatory; the working topics here use plain `variable: Topic.X`. Both are valid, and the file's existing style is proven.
- **`Copilot Studio: Reattach Agent` is the missing first step for anyone who gets these agents via Git.** Cloning the repo yields agent folders with no binding to an environment; Preview/Get/Apply do nothing until Reattach runs. Also note Apply updates the live agent immediately but does not publish, Apply is blocked while unretrieved remote changes exist, and Get overwrites uncommitted local work, so commit before Get.
